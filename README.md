# Codex学習Wiki

外部AI APIを使わず、Codexがローカルファイル `data.js` を編集して文書を増やす学習Wikiです。

## 使い方

1. `index.html` をブラウザで開く。
2. 本文中の分からない語句を範囲選択する。
3. 選択範囲の上で右クリックする。
4. 画面に強調された `Codex Request` が表示され、URLにも生成依頼が入る。
5. `依頼文をコピー` を押す。コピー成功時は画面に `コピーしました` と表示される。ブラウザ制約で自動コピーできない場合は、依頼文が選択されるので `Ctrl+C` でコピーする。
6. このCodexチャットに「生成して」と送る。
7. Codexが `data.js` に文書を追加する。
8. ブラウザをリロードすると新しい文書が開ける。

## スマホで見る

PCとスマホを同じWi-Fiに接続し、PC側で `start-phone-server.ps1` を実行します。

表示された `http://PCのIP:8080/index.html` をスマホのブラウザで開くと、`data.js` に入っている全文書を読めます。

スマホで開けない場合は、PCとスマホが同じWi-Fiか確認し、Windows Defender Firewall の確認が出たら Python の通信を許可してください。

停止するときは `stop-phone-server.ps1` を実行します。

## 同じWi-Fi外から見る

同じWi-Fi外から見る場合は、`build-public-site.ps1` で公開用の `public_site` フォルダを作り、無料の静的ホスティングへアップロードします。

詳しくは `PUBLIC_HOSTING.md` を参照してください。

知らない人に読まれたくない場合は、平文の `data.js` を公開する `public_site` ではなく、暗号化版の `private_site` を使ってください。

詳しくは `PRIVATE_HOSTING.md` を参照してください。

Netlifyで自動更新したい場合は、GitHubとNetlifyを接続します。詳しくは `NETLIFY_AUTO_DEPLOY.md` を参照してください。

## 重要な設計

- APIキーは不要です。
- ブラウザだけでAI生成はしません。
- AI生成はCodex上で行い、結果を `data.js` に保存します。
- 同じ要素名が複数文書に出た場合、同じ文書へリンクされるようにします。
- 既に文書がある要素は、クリックすると生成せず既存文書を開きます。
- 本文中の文字を範囲選択して作った文書は、生成後に `parentLinks` をもとに親本文内の該当語句がクリック可能になります。

## データ

現在の文書データは `data.js` に保存されています。題材入力や本文選択からCodex生成依頼を作り、必要な文書を追加していきます。
